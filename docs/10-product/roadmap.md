# 版本路线

## V0 — 验证原型

目标：验证需求和交互思路

- AO3 模拟页面
- hover 按钮 + 四种操作
- 假数据反馈
- 不要求真实插件能力

## V1 — MVP

目标：跑通桌面端 Chrome 闭环

- 页面 tag 识别（列表页 + 作品详情页）
- 三种匹配方式 + 四类规则
- 页面快速添加 + 即时重渲染
- Popup 当前页统计 + 全局开关
- Options 基础规则管理
- 本地存储
- 折叠 / 警示 / 高亮 / 弱化视觉表现

## V1.1 — 易用性增强

- 规则导入 / 导出 JSON
- 临时停用（当前页 / 某规则组）
- 当前页命中原因解释
- 简单规则分组
- 一键从当前 tag 进入高级规则创建（选匹配方式 / 作用范围 / 规则组）

## V1.5 — 规则系统升级

- 规则优先级手动编辑
- 更丰富匹配逻辑（多关键词组合、与/或条件、排除条件）
- 作用范围扩展（标题、简介、系列信息）
- 页面维度规则（仅在特定 fandom 页面生效）
- 规则使用统计

## V2 — 跨设备 / 跨端

前提：桌面端留存成立、规则体系稳定、用户确实需要更多平台支持。

- 多套主题样式 + 样式强度配置
- 更优雅的折叠体验（批量计数、一键展开）
- 撤销操作
- 规则模板 / 社区分享
- 跨浏览器迁移（Firefox → Safari）
- 移动端交互层重做（tap / long press / action sheet）
