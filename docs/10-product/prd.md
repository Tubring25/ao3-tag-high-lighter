# 产品需求文档 — AO3 Tag Highlighter

## 问题

AO3 作品携带大量 tag（relationship / character / freeform），一篇 fic 常常几十个。用户面临三个反复出现的痛点：

1. **扫读成本高。** 大多数 tag 和当前读者无关，真正在意的几个淹没在噪音里。
2. **关键信息容易漏。** 雷点 tag（trigger、squick、不想看的 trope）混在列表中，导致误点、误读。
3. **现有解决方式笨重。** AO3 自带过滤太粗、userscript 门槛高、site skin 不够即时，都不是产品化的体验。

## 目标

让用户通过自定义 tag 规则，实时改变 AO3 页面的视觉呈现，从而更快做出"读还是跳过"的决策。

具体来说：

- **加速扫读** — 让每个用户在意的 tag 一眼可见。
- **个性化视图** — 同一个 AO3 页面，不同用户看到不同的视觉强调。
- **低成本维护规则** — 看到一个 tag → 点一下 → 规则就建好了。

## 用户类型

| 类型 | 行为特征 |
|---|---|
| Tag 驱动型读者 | 主要靠 pairing、trope、warning、AU 等 tag 决定是否点进去 |
| 雷点敏感型读者 | 需要快速发现风险 tag，希望直接折叠或标红相关作品 |
| 高频浏览型读者 | 大量刷列表页，希望降低认知疲劳 |
| 愿意轻度配置的用户 | 愿意维护 10–50 条规则，不排斥插件设置 |

## 用户故事

1. **作为浏览 tag listing 的读者，** 我希望喜欢的 trope tag 被高亮，这样我一眼就能发现值得看的 fic。
2. **作为有雷点的读者，** 我希望含有雷点 tag 的作品被警示或折叠，这样我不会误点进去。
3. **作为浏览搜索结果的读者，** 我希望噪音 tag（作者碎碎念、无意义 freeform）被弱化，这样重要 tag 更突出。
4. **作为在页面上悬停任意 tag 的读者，** 我希望出现一个快速按钮，让我不离开页面就能添加规则。
5. **作为回访用户，** 我希望规则跨会话持久化，只需设置一次。

## 支持的页面

- 搜索结果页
- Tag 浏览 / listing 页
- Fandom 作品列表页
- Bookmarks 列表页
- 其他 works 列表型页面
- 单篇作品详情页（tag 区域）

## 功能需求

### Tag 识别

- 从每个作品卡片 / 作品详情中解析 tag。
- 分类为：**relationship**、**character**、**freeform**。
- 每个 tag 关联其所属作品。

### 规则模型

每条规则包含：

- **Pattern** — 要匹配的文本。
- **Action** — `highlight` | `warn` | `mute` | `hideWork` 四选一。
- **Match mode** — `exact`（精确）| `contains`（包含）| `wildcard`（通配符，支持 `*`）。
- **Category scope** — 作用于 relationship / character / freeform / all。
- **Enabled** — 可启停，不必删除。

### 匹配与优先级

同一 tag 命中多条规则时，按 action 权重从高到低取胜：

1. `hideWork`（最高 — 影响整个作品是否可见）
2. `warn`（影响作品级别的风险呈现）
3. `highlight`（tag 级正向强调）
4. `mute`（tag 级弱化）

### 规则效果 — 视觉表现

| Action | 作用对象 | 视觉效果 |
|---|---|---|
| `highlight` | Tag | 正向样式强调（如彩色背景） |
| `mute` | Tag | 降低透明度 / 灰化 |
| `warn` | 作品卡片 | 作品级警告边框或横幅 |
| `hideWork` | 作品卡片 | 折叠为一条细占位条；点击可展开 |

### 页面交互

- **悬停按钮** — 鼠标悬停任意 tag 时出现小按钮，点击打开快速添加菜单。
- **快速添加菜单** — 选择 action，自动填充 tag 文本，一键创建规则。
- **即时重渲染** — 规则添加后页面立即更新。
- **Toast 提示** — 规则创建后短暂确认提示。

### Popup（扩展弹窗）

- 显示当前页命中摘要（按 action 类型统计数量）。
- 全局开关。
- 跳转到 options 页面的入口。

### Options（设置页）

- 完整规则列表，支持搜索 / 筛选。
- 新增 / 编辑 / 删除规则。
- 单条规则启停。

### 存储

- 全部数据存于 `chrome.storage.local`。
- 不发网络请求、不登录、不云同步。
