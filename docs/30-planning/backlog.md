# 开发任务清单

## 优先级说明

- **P0** — 必须先做，不做插件跑不起来
- **P1** — 建议一起做，不做也能跑但体验明显差
- **P2** — 后置到 V1.1，先不做也没问题

---

## P0：核心闭环

### 模块 A：Manifest 与基础插件壳

| ID | 任务 | 验收标准 |
|---|---|---|
| A1 | 建立 MV3 项目，配置 TypeScript + Vite，输出 manifest / content / popup / options / background | 插件能在 Chrome 开发者模式下加载，popup 能打开，content script 能注入 AO3 |
| A2 | 配置最小权限（storage, scripting, host_permissions: archiveofourown.org） | 权限声明正确，无多余权限 |

### 模块 B：AO3 页面解析

| ID | 任务 | 验收标准 |
|---|---|---|
| B1 | 识别 AO3 页面类型（列表页 / 作品详情页） | 能区分支持的页面 vs 不支持的页面 |
| B2 | 解析作品容器 + tag 列表 | 能提取 ParsedWork[] 和 ParsedTag[] |
| B3 | 区分 tag 类型：relationship / character / freeform | 每个 tag 正确分类 |
| B4 | tag 文本标准化 | 大小写、空格等不影响匹配 |

### 模块 C：规则引擎

| ID | 任务 | 验收标准 |
|---|---|---|
| C1 | 定义 Rule / Settings / MatchResult 类型 | types.ts 编译通过 |
| C2 | 实现 exact 匹配 | 精确匹配通过测试 |
| C3 | 实现 contains 匹配 | 包含匹配通过测试 |
| C4 | 实现 wildcard 匹配 | `*` 通配符通过测试 |
| C5 | 实现 category 过滤 | 规则只在指定 tag 类型上生效 |
| C6 | 实现优先级解析 | hideWork > warn > highlight > mute |
| C7 | disabled 规则不参与匹配 | 启停逻辑正确 |

### 模块 D：页面渲染

| ID | 任务 | 验收标准 |
|---|---|---|
| D1 | highlight 样式 | 命中 tag 有正向视觉强调 |
| D2 | mute 样式 | 命中 tag 降低透明度 / 灰化 |
| D3 | warn 样式 | 命中作品显示警告边框或横幅 |
| D4 | hideWork 样式 | 命中作品折叠为占位条，点击可展开 |
| D5 | 重算时不重复插入 DOM | 清理旧样式后再渲染 |

### 模块 E：本地存储

| ID | 任务 | 验收标准 |
|---|---|---|
| E1 | 规则 CRUD 封装 | 新增 / 读取 / 更新 / 删除规则正常 |
| E2 | 设置读写封装 | Settings 持久化和读取正常 |
| E3 | 数据校验 | 写入前校验必填字段和格式 |

### 模块 F：页面交互 — 悬停菜单

| ID | 任务 | 验收标准 |
|---|---|---|
| F1 | tag hover 时显示小按钮 | 按钮位置正确，不遮挡 tag 文本 |
| F2 | 点击按钮弹出菜单 | 菜单内含四种 action 选项 |
| F3 | 选中 action → 创建规则 | 规则写入 storage |
| F4 | 创建后即时重渲染 | 页面立即反映新规则效果 |

### 模块 G：Popup

| ID | 任务 | 验收标准 |
|---|---|---|
| G1 | 显示当前页命中统计（各 action 数量） | 数据准确 |
| G2 | 全局开关 | 开关状态持久化，关闭后 content script 停止渲染 |
| G3 | 跳转到 options 页面 | 链接正常 |

### 模块 H：Options 页面

| ID | 任务 | 验收标准 |
|---|---|---|
| H1 | 显示全部规则列表 | 列表正确渲染 |
| H2 | 搜索 / 筛选规则 | 能按 pattern 文本搜索 |
| H3 | 手动新增规则 | 填写表单 → 保存成功 |
| H4 | 编辑规则 | 修改字段 → 保存成功 |
| H5 | 删除规则 | 删除后列表更新 |
| H6 | 启停单条规则 | toggle 状态持久化 |

## P1：体验补全

| ID | 任务 | 验收标准 |
|---|---|---|
| I1 | Toast 反馈 | 规则创建 / 删除后显示短暂提示 |

## 推荐开发顺序

### 第 1 阶段：跑通最小闭环

模块 A + B + C + D + E（写死几条测试规则）

**完成标志：** 不用任何 UI，打开 AO3 页面就能看到规则效果。

### 第 2 阶段：页面内交互

模块 F + I1

**完成标志：** 用户不进设置页，也能从页面直接建立规则。

### 第 3 阶段：管理界面

模块 G + H

**完成标志：** 已有规则可以完整维护。

### 第 4 阶段：补稳定性

I2–I6

**完成标志：** 可以给真实用户装起来用。
